本物の.MPPバイナリを読む

04.MPPファイルを開く — よくある質問

.MPPファイルを開くには？

最も速い方法はIngamttでファイルを開くことです。.MPPはMicrosoft Projectのネイティブバイナリ形式で、デフォルトではMicrosoft Projectだけがきれいに開けますが、MS ProjectはWindows専用で高額、一度きりのファイルのために購入するには向きません。Ingamttは.MPPを直接インポートし、Mac、iPhone、iPad、Windows、Android、ブラウザで動作。7日間の無料体験（カード不要）で約30秒で完全なプロジェクトが表示されます。

Microsoft ProjectなしでMacで.MPPファイルを開く方法は？

Microsoft ProjectはmacOSでは動作しません。MacでMS Projectを使うことはできないのです。したがって答えは「MacでProjectを動かす」ではなく「.MPPをネイティブに読むツールを使う」です。IngamttはMac App Storeにネイティブアプリを提供し、Apple SiliconおよびIntel MacでDirect.MPPファイルを開きます。タスク、依存関係、クリティカルパスが完全に保持されます。ブラウザ版も同じように機能します。

iPhoneやiPadで.MPPファイルを開けますか？

はい。IngamttはApp StoreにiPhoneとiPad用のネイティブiOSアプリを提供しており、.MPPとMS Project XMLファイルを直接開きます。iPadではピンチズームとタッチスクロール付きの広いタイムラインが表示され、iOSデバイスでのMicrosoft Projectデスクトップ体験に最も近いものが得られます。同僚からメールで送られてきた.MPPを開く最速の方法です。

依存関係、リソース、クリティカルパスは失われますか？

Ingamttでは失われません。重要なフィールドについて、インポートは損失なしです。タスク、サブタスク、先行-後続依存、リソース割り当て、作業と期間、マイルストーン、クリティカルパス計算がすべて元の.MPPファイルから取り込まれます。Ingamttは内部的にMicrosoft Projectと同じプロジェクト構造を使用します。ガントチャートを見るだけでよい場合は、ガントビューアページをご覧ください。
